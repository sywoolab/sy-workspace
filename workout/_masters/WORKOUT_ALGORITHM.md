# 운동 알고리즘

> WORKOUT_MASTER.md의 규칙/원칙 하위에서 동작한다. 의사판단 알고리즘은 WORKOUT_MASTER.md를 따른다.
> 이 파일은 데이터 분석, 스케줄 조정, 목표 추적 로직만 다룬다.

## 이론적 기반

### 적용 방법론
| 출처 | 적용 원리 |
| ------ | ------ |
| Joe Friel (Triathlete's Training Bible) | 주기화 모델: Base → Build → Peak, 매 3-4주마다 회복 주 |
| Matt Fitzgerald / Stephen Seiler | 80/20 강도 배분: 훈련의 80% 저강도(Zone 1-2), 20% 고강도(Zone 4-5) |
| Jack Daniels (Running Formula) | VDOT 기반 훈련 페이스 존 설정 |
| Banister Fitness-Fatigue Model | 누적 피로 관리: Fitness 감쇠 ~50일, Fatigue 감쇠 ~11일 |
| Bosquet et al. (2007) 메타분석 | 테이퍼: 2주, 볼륨 41-60% 감소, 강도·빈도 유지, 퍼포먼스 ~3% 개선 |

### 핵심 원칙
1. **80/20 강도 배분**: 주간 훈련 시간의 80%는 Easy(Zone 1-2), 20%만 Tempo 이상. "쉬운 날은 진짜 쉽게, 빡센 날은 진짜 빡세게" (Seiler)
2. **회복 주 필수**: 50세 미만은 매 4주마다, 50세 이상은 매 3주마다 회복 주 배치 (Friel). 회복 주 = 볼륨 40-50% 감소, 강도 유지
3. **훈련 페이스 ≠ 레이스 페이스**: 레이스 페이스로 뛰려면, 훈련 템포는 그보다 15-17초/km 빠르게 (Daniels VDOT)
4. **브릭 적응은 반복으로만**: 아마추어 브릭 감속률 5-8%, 전환기 ~679m. 반복 훈련으로만 줄일 수 있음 (PMC6572577)
5. **점진적 과부하**: 주간 볼륨 증가 최대 10%. 급격한 증가 = 부상

## 대회 목표

### 목표 시간: 2:50 (170분)

근거: 남성 35-39세 올림픽 디스턴스 평균 ~2:47, 첫 참가자 평균 3:00-3:30.
사이클 선수 출신 + FM 4:00/HM 1:35 이력 → 자전거에서 대폭 시간 절약 가능. **2:50은 공격적이지만 달성 가능한 목표.**

### 러닝 페이스 역산 (핵심)

**레이스 당일 러닝 10km 58분(5:48/km)을 달성하려면:**

```
레이스 브릭 런 페이스: 5:48/km
↑ 브릭 감속 보정 (-7%, PMC6572577 아마추어 기준)
단독 레이스 페이스: 5:24/km (= 10km 54분)
↑ 테이퍼 효과 (-3%, Bosquet 메타분석)
테이퍼 전 실력: 5:34/km (= 10km 55:40)
↑ 이것이 Phase 2 종료 시 도달해야 할 단독 10km 페이스
```

**따라서 훈련 페이스 존 (VDOT ~36, 10km 55분 기준):**

| 존 | 페이스 | 심박 존 | 주간 비율 | 용도 |
| ------ | ------ | ------ | ------ | ------ |
| **Easy (E)** | 5:55~6:42/km | Zone 1-2 (HR < 145) | **80%** | 롱런, 회복런, 워밍업/쿨다운 |
| **Marathon (M)** | 5:24/km | Zone 2-3 | 간헐적 | 마라톤 페이스 감각 |
| **Tempo (T)** | 5:10~5:15/km | Zone 3-4 (HR 155-165) | **15%** | 화요일 템포런, 20-30분 지속 |
| **Interval (I)** | 4:47/km | Zone 4-5 (HR 165-175) | **5%** | VO2max 인터벌 (Phase 2부터) |
| **Repetition (R)** | 4:23/km | Zone 5 | 극소 | Phase 2 후반 스피드 터치 |

**80/20 적용 예시 (주간):**
- 주 3회 러닝 중 → Easy 2회 + Tempo/Interval 1회
- Easy 런: 반드시 6:00/km 이상으로. "너무 느린 거 아닌가?" 싶을 정도가 맞다 (Fitzgerald)
- "5:30/km으로 쉽게 뛰었어요" → 이것은 Easy가 아니라 Moderate, 가장 비효율적인 존

## 누적 피로 모델 (Fitness-Fatigue)

### 일일 훈련 부하 (Training Load) 산정

| 종목 | 부하 산정 기준 | 예시 |
| ------ | ------ | ------ |
| 수영 | 시간(분) × 강도계수 | 30분 Easy = 30, 30분 Hard = 45 |
| 러닝 | 시간(분) × 강도계수 × 1.3 (충격 보정) | 40분 Easy = 52, 40분 Tempo = 78 |
| 자전거 | 시간(분) × 강도계수 × 0.8 (저충격) | 60분 Easy = 48, 60분 Hard = 72 |

강도계수: Easy=1.0, Moderate=1.2, Tempo=1.5, Hard=1.8

### 수영 장비별 강도 보정

수영 수업(월/수/금) 및 개인교습(토)에서 보조장비 사용 시, 페이스와 강도 해석을 보정한다.
사용자가 장비를 알려주거나, 심박이 비정상적으로 낮으면 장비 사용을 유추한다.

| 장비 | 페이스 영향 | 강도 보정 | 비고 |
| ------ | ------ | ------ | ------ |
| 맨몸 (no equipment) | 기준 | ×1.0 | 대회 조건과 동일. 실전 페이스 평가에 사용 |
| 오리발 (fins) | 빠름 -10~15초/100m | ×0.7 | 하체 부담 감소, 심박 낮음. 페이스를 맨몸으로 환산 시 +12초 |
| 패들 (paddles) | 빠름 -5~8초/100m | ×1.2 | 어깨·상체 부하 증가. 기술 훈련 효과 |
| 풀부이 (pull buoy) | 빠름 -3~5초/100m | ×0.8 | 하체 비활성, 상체 집중. 심박 낮음 |
| 오리발 + 패들 | 매우 빠름 | ×0.9 | 상하체 분산. 페이스 해석 무의미 |
| 킥보드 (kickboard) | 매우 느림 | ×0.9 | 하체 전용 드릴. 심폐보다 근력 훈련 |

**적용 규칙:**
1. **실전 페이스 평가**: 맨몸 세션 데이터만 사용. 장비 착용 세션의 페이스는 벤치마크 비교에서 제외
2. **강도 계산**: `수영 부하 = 시간 × 강도계수 × 장비보정`
3. **장비 판별**: 사용자가 명시하지 않아도, 동일 노력 대비 페이스가 갑자기 10초+ 빨라지거나 심박이 평소 대비 -15% 이상이면 장비 사용 가능성을 질문
4. **주간 맨몸 최소**: 벤치마크 평가를 위해 주 1회 이상 맨몸 수영 권장

### 주간 피로 관리

| 지표 | 녹색 | 황색 | 적색 |
| ------ | ------ | ------ | ------ |
| 주간 총 부하 | 목표의 90-110% | 80-90% 또는 110-120% | <80% 또는 >120% |
| 일일 최대 부하 | < 주간평균 × 1.5 | 1.5~2.0배 | > 2.0배 |
| 연속 고강도 일수 | 0-1일 | 2일 | 3일+ (금지) |
| 안정시 심박 변동 | 기준선 ± 5bpm | ± 5-10bpm | > +10bpm (과훈련 신호) |

### 회복 주 (Recovery Week) 프로토콜

- **타이밍**: 매 4주마다 (Week 3 → 회복, Week 7 → 대회 주)
- **볼륨**: 평상시의 50-60%
- **강도**: 유지하되 고강도 세션 1회로 축소
- **러닝**: Easy only, 거리 50% 감소
- **수영**: 기술 드릴 중심
- **자전거**: 가볍게 30-40분
- 회복 주 첫 1-3일: Zone 1만 (Friel)

### 테이퍼 프로토콜 (대회 2주 전, 4/27~5/9)

| 항목 | 방법 | 근거 |
| ------ | ------ | ------ |
| 볼륨 | **지수적 감소**, 50% → 25% | Bosquet 메타분석: 41-60% 감소 최적 |
| 강도 | **유지** — 줄이지 않음 | 강도 유지가 체력 유지의 핵심 |
| 빈도 | **유지** — 줄이지 않음 | 빈도 감소는 기술 감각 손실 |
| 1주차 (4/27-5/3) | 평상시 볼륨의 60% | 지수적 감소 초기 |
| 2주차 (5/4-5/8) | 평상시 볼륨의 30% | 급격 감소, 피로 해소 |
| 기대 효과 | 퍼포먼스 **~3% 개선** | 10km 57분 → 55분 수준 |

## Phase별 벤치마크 (재설계)

### Phase 1: 베이스 (3/16 ~ 4/5) — 3주

**목적**: 러닝 빈도 확보, 유산소 기반 재구축, 자전거 감각 회복
**강도 배분**: Easy 90% / Tempo 10% (페이스 추구 금지)

| 종목 | 벤치마크 | 왜 이 수준인가 |
| ------ | ------ | ------ |
| 러닝 빈도 | 주 3회 | 빈도 확보가 최우선. 페이스는 무관 |
| 러닝 롱런 | 10km 논스톱 (6:00~6:30 Easy) | 80/20: Easy로 거리만 늘려라 |
| 러닝 주간 볼륨 | 20km+ | |
| 수영 | 풀 1:55/100m 이하 유지 | 현재 1:57, 미세 개선 |
| 자전거 | 60분 완주, 에어로 30분 | 감각 회복 |
| 브릭 | 자전거 후 2~3km 조깅 (페이스 무관) | 전환 신경근 적응 시작 |
| 회복 주 | Week 3 (3/30~4/5) = 볼륨 50% | 수퍼컴펜세이션 확보 |

**Easy 러닝 페이스 가이드**: 6:00~6:42/km. 대화 가능한 속도. 이보다 빠르면 Moderate 존에 빠진 것.

### Phase 2: 빌드 (4/6 ~ 4/26) — 3주

**목적**: 러닝 페이스 끌어올리기, 브릭 적응, 레이스 시뮬레이션
**강도 배분**: Easy 80% / Tempo 15% / Interval 5%

| 종목 | 벤치마크 | 왜 이 수준인가 |
| ------ | ------ | ------ |
| 러닝 템포 | 3km @ 5:10~5:15/km | VDOT 36 Threshold = 5:15. 이 페이스 유지 = 10km 55분 수준 |
| 러닝 Easy | 반드시 6:00+/km | 80/20 준수. 쉬운 날 빠르게 뛰면 피로만 누적 |
| 러닝 단독 10km | 55~56분 (5:30~5:36/km) | 8주 복귀 현실적 도달치 (VO2max 85-90% 회복) |
| 수영 | 풀 1:50/100m + 사이팅 체화 | |
| 수영 OW | 아쿠아슬론(4/18) 완주 | OW 첫 경험 |
| 자전거 | 75-90분, 평균 30km/h+ | |
| 브릭 풀 | 자전거 60분 → 러닝 5km @ 5:50~6:10 | 브릭 감속 적응. 첫 2km은 느려도 OK (전환기 679m) |
| 회복 주 | Week 7 (5/4~5/9) = 대회 주 (테이퍼) | |

**화요일 템포런 구조** (Jack Daniels 방식):
```
워밍업 2km Easy (6:00+)
→ 템포 3km @ 5:10~5:15/km (Threshold)
→ 쿨다운 2km Easy (6:00+)
= 총 7km, 그 중 고강도는 3km (43%)만
```

### Phase 3: 테이퍼 (4/27 ~ 5/9) — 2주

**목적**: 피로 해소 + 레이스 감각 유지
**강도 배분**: Easy 80% / 레이스 페이스 터치 20% (볼륨은 50% → 25%)

| 종목 | 벤치마크 | 왜 이 수준인가 |
| ------ | ------ | ------ |

| 수영 | 1km 가볍게 + 사이팅 | 감각 유지 |
| 자전거 | 30분 가볍게 | |
| OW | 5/8 수성못 300m 사전 입수 | 수온·시야 확인 |
| 테이퍼 효과 | 퍼포먼스 ~3% 개선 기대 | 10km 56분 → 54분 수준 |

## 예상 완주시간 계산 모델 (개선)

### 기존 문제점
- 최근 페이스 × 거리 = 기계적 산술
- 브릭 감속 +30초 고정값
- 테이퍼·적응·레이스데이 효과 무시

### 개선된 모델

```
예상 완주시간 = 예상수영 + T1 + 예상자전거 + T2 + 예상러닝(브릭)

■ 예상수영 (OW)
  = (최근 풀 페이스 + OW_보정) × 15
  OW_보정:
    - 사전 OW 경험 0회: +17초/100m
    - 사전 OW 경험 1-2회: +12초/100m
    - 사전 OW 경험 3회+: +8초/100m

■ 예상자전거
  = 40km ÷ 예상평균속도
  데이터 있으면 최근 3회 평균, 없으면 32km/h (사이클 선수 기본값)

■ 예상러닝 (브릭) — 역산 모델
  단독_10km_페이스 = 최근 러닝 데이터 기반 VDOT → 10km 예측
  브릭_감속 = 단독 페이스 × 감속률
    - 브릭 훈련 0-2회: 8% 감속
    - 브릭 훈련 3-5회: 6% 감속
    - 브릭 훈련 6회+: 5% 감속
  테이퍼_효과 = -3% (대회 2주 전 적절한 테이퍼 가정)

  예상_브릭_페이스 = (단독_페이스 × (1 + 감속률)) × (1 - 테이퍼효과)
  예상러닝 = 예상_브릭_페이스 × 10
```

### VDOT 기반 러닝 예측 (lookup table)

최근 러닝 데이터에서 VDOT를 추정하고, 10km 레이스 타임을 예측한다.

| 최근 데이터 예시 | 추정 VDOT | 10km 예측 | 브릭 10km 예측 (7% 감속) |
| ------ | ------ | ------ | ------ |
| 5km @ 5:33/km | ~35 | 58:00 | 62:04 |
| 7km @ 5:30/km | ~36 | 56:30 | 60:28 |
| 10km @ 5:24/km | ~37 | 55:00 | 58:51 |
| 10km @ 5:12/km | ~38 | 53:30 | 57:15 |
| 10km @ 5:00/km | ~40 | 51:00 | 54:34 |

## 운동 데이터 입력 프로토콜

사용자가 운동 데이터(사진, 텍스트, 가민 스크린샷 등)를 제공하면:

### Step 1: 데이터 파싱
1. 종목 식별 (수영/러닝/자전거/브릭)
2. 핵심 지표 추출:
   - 수영: 총 시간, 이동 시간, 페이스(/100m), 평균HR, 최대HR, 거리, Swolf
   - 러닝: 거리, 페이스(/km), 평균HR, 최대HR, 케이던스, **훈련 존 판정**
   - 자전거: 시간, 거리, 평균속도, 평균HR, 파워(있으면)
3. workout_log.json 업데이트
4. **훈련 존 판정**: 실제 뛴 페이스가 Easy/Tempo/Interval 중 어디에 해당하는지 태그

### Step 2: 분석
1. **VDOT 업데이트**: 러닝 데이터가 들어올 때마다 추정 VDOT 재계산
2. **80/20 준수 확인**: 이번 주 Easy vs 고강도 비율 체크
3. **누적 부하 계산**: 일일 부하 × 잔존 피로 (Fatigue 감쇠 11일)
4. **Phase 벤치마크 대비 진척도**
5. **페이스 트렌드**: 최근 3회 동일 종목 비교
6. **브릭 적응도**: 브릭 훈련 누적 횟수 → 감속률 보정

### Step 3: 스케줄 조정 판단
- 자동 조정 규칙 적용
- 필요 시 workout_schedule.json overrides 업데이트

### Step 4: 텔레그램 메시지 전송
workout_log.json 업데이트 + git push → GitHub Action 자동 실행

메시지 포맷:
```
🏋️ 운동 기록 업데이트

📊 오늘의 운동
{종목} | {거리/시간} | {페이스} | HR {평균}/{최대}
훈련 존: {Easy/Tempo/Interval} {80/20 준수 여부}

📈 Phase {N} 진척 (VDOT {추정값})
러닝: {현재 10km 예측} → 목표 {Phase 벤치마크} {✅/⚠️}
수영: {현재 페이스} → 목표 {벤치마크} {✅/⚠️}
브릭 적응: {누적 횟수}회 (감속률 {N}%)

📅 이번 주 현황
수영 ●○○○ | 러닝 ●○○ ({km}km) | 자전거 ○
부하: {현재}/{목표} | 80/20: {Easy%}/{고강도%}

🔄 스케줄 조정
{변경 내용 + 사유, 또는 "변경 없음"}

🏁 D-{N} | 목표 2:50 | 예상 {시간} {🟢/🟡/🔴}
  수영 {분}분 + 자전거 {분}분 + 러닝 {분}분 (브릭 보정 {N}%)
```

### Step 5: user_fitness.md 업데이트
러닝 기록 추이, 수영 기록에 신규 데이터 추가
